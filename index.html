Projekty – wizytówka umiejętności

Kliknij na kafelek, aby zobaczyć szczegóły architektury i działania

CloudBudget – FinOps

Platforma agregacji kosztów chmury z prognozowaniem trendów i rekomendacjami oszczędności.

PythonFastAPICeleryDuckDB

Architektura CloudBudget

Jak działa: Aplikacja pobiera dane kosztowe z API (symulacja), zapisuje do PostgreSQL. Zadania asynchroniczne (Celery+RabbitMQ) uruchamiają agregację i prognozowanie. DuckDB wykonuje szybkie zapytania analityczne na dużych zbiorach (do 10GB). Wyniki cache'owane w Redis. Frontend w FastAPI + Jinja2, dashboard z Chart.js.

Technologie: FastAPI, SQLAlchemy, Celery, Redis, DuckDB, Docker Compose, Pytest. Skalowalność: workerzy Celery rozproszeni, możliwość dodania wielu instancji.

Umiejętności: projektowanie asynchronicznych przepływów, integracja baz danych, optymalizacja zapytań OLAP, konteneryzacja.

InfraFlow – Monitoring

System zbierania metryk, dashboard Grafana i alerty. Obserwowalność infrastruktury.

PythonTimescaleDBRedisGrafana

InfraFlow – szczegóły techniczne

Budowa: Agent napisany w Pythonie (psutil, socket) wysyła metryki co 10s do API. API zapisuje dane w TimescaleDB (baza szeregów czasowych). Redis Streams buforuje zdarzenia przy dużym ruchu. Integracja z Prometheus (exporter) i wizualizacja w Grafanie. Alerty wysyłane przez webhook.

Zastosowane wzorce: Repository pattern, asynchroniczne przetwarzanie kolejek. Zapewnia retencję danych przez polityki kompresji TimescaleDB.

Co pokazuje: Znajomość monitoringu, pracy z danymi time-series, Docker Compose do uruchomienia całego stacku (Grafana, Prometheus, TimescaleDB).

NetGuardian – SecOps

Platforma do analizy logów bezpieczeństwa, detekcji anomalii z użyciem Apache Kafka.

KafkaWebSocketsDuckDB

NetGuardian – architektura strumieniowa

Przepływ danych: Producerzy wysyłają logi (syslog, custom) do topiku Kafka. Konsumenci w Pythonie (aiokafka) przetwarzają w czasie rzeczywistym – silnik reguł (YARA-like) skanuje zdarzenia. Wykryte incydenty trafiają do bazy oraz WebSocket → powiadomienia na dashboardzie.

Skalowalność: Partycje Kafka, grupy konsumentów. DuckDB używana do analizy historycznej. Dashboard pokazuje metryki zdarzeń na sekundę oraz geoIP (MaxMind).

Umiejętności: stream processing, bezpieczeństwo sieciowe, projektowanie systemów event-driven.

NetAegis – Orchestration

Agentowa orkiestracja konfiguracji sieci (MCP) z historią zmian.

FastAPIParamikoRedis

NetAegis – MCP i automatyzacja

Architektura: API centralne (FastAPI), agent NetConfig (Paramiko) do ssh na urządzeniach Cisco/ Linux. Kolejka zadań RQ (Redis). NetPulse zbiera telemetrię, SecLog agreguje logi. Wszystkie akcje są rejestrowane w SQLite z wersjonowaniem konfiguracji.

Przykład użycia: Operator wysyła żądanie zmiany VLAN przez API – system waliduje, umieszcza w kolejce, agent wykonuje i zwraca status. Nieudane zmiany automatycznie rollback.

Co demonstruje: Automatyzacja sieci, idempotentność, bezpieczeństwo zmian, praca z urządzeniami poprzez SSH.
